# Domain Docs

This repo uses a single domain context.

## Before Exploring

Read these when planning, creating PRDs, creating issues, diagnosing bugs, or reviewing architecture:

- `CONTEXT.md` at the repo root for canonical Boardsmith product language.
- `docs/adr/` if it exists and an ADR touches the area being changed.
- Product planning docs relevant to the task, especially `PRODUCT.md`, `docs/VISION.md`, `DESIGN.md`, `docs/WEBSITE_PLANNING_PROCESS.md`, and the active PRD.

If an optional ADR directory does not exist, proceed silently.

## Vocabulary Rule

When output names a domain concept, use the term as defined in `CONTEXT.md`. Do not drift to synonyms the glossary explicitly avoids.

## ADR Conflicts

If a proposal contradicts an existing ADR, surface the contradiction explicitly instead of silently overriding it.
